项目环境搭建

安装jdk1.8

安装docker（CentOS）

安装mysql

安装redis

Maven

安装开发插件(可选-方便开发)

安装git

创建仓库

在码云新建仓库，仓库名gulimall，选择语言java，在.gitignore选中maven， 许可证选Apache-2.0，开发模型选生产/开发模型，开发时在dev分支， 发布时在master分支，创建如图所示

设置仓库开源

管理 > 基本信息

新建项目

在IDEA中使用git新建项目

创建模块

商品服务product

存储服务ware

订单服务order

优惠券服务coupon

用户服务member

每个模块导入web和openFeign

新建pom父级工程文件，聚合模块

修改各模块pom文件，继承父工程pom文件

创建数据库

gulimall-oms.sql

gulimall-pms.sql

gulimall_sms.sql

gulimall_ums.sql

gulimall_wms.sql

注意：不能将对应sql文件拉进点开的数据库自动运行sql语句，因为sql文件编码是gbk，直接拖进去会导致中文乱码

快速构建后台管理系统

码云上搜索人人开源，找到renren-fast(后台管理系统的后端)，renren-fast-vue(后台管理系统的前端), 克隆到本地。

人人开源

克隆：

桌面右键git bash克隆两个项目：

删掉两个文件夹里的.git文件夹

添加到项目模块

把renren-fast复制到子模块，并加入父工程

把后端renren-fast文件夹复制到谷粒商城项目中并加入父工程

parent继承父工程

renren-fast的pom修改

  • pom.xml报错

报错原因：子模块的parent写的并不是子模块的上一级，而是继承了springboot

解决方法：将其他模块pom的parent复制粘贴到renren-fast的pom

parent继承父工程

  • 如果pom.xml之中的plugin地方报错，引入相关依赖

创建后台管理系统的数据库gulimall_admin

新建数据库gulimall_admin

sql文件在renren-fast/db/ 目录下，找到其中的mysql.sql 复制，运行。

修改配置文件

修改renren-fast的配置文件，设置数据库

查看application.yml，发现当前环境为dev

修改application-dev.yml, 配置成自己的linux的数据库信息，数据库名为gulimall_admin、账号密码：

如果报错ssl错误，在url后面加&useSSL=false

配置语言级别为8

配置Java编译器

file - > settings - > java complier —>找到项目模块

将目标字节码版本 全都改成 1.8 即可

启动和访问

运行项目，访问地址

报错解决

原因：
Spring Boot版本升级后lombok版本相关api有所改动

先修改pom.xml文件中lombok版本

修改CorsConfig.java文件

再次运行成功
